CBT5 全面流程化设计

🎯
CBT5 目标：TAPD 覆盖「总单 → 策划 → 开发 → 测试」全链路，让每个需求从诞生到上线的每一步都有迹可循。

2.1 四阶段流程

1 策划阶段 — 总单驱动

TAPD操作：策划创建【总单】（如"家族系统-任务-兑换商店"），在总单下拆分【子需求】（策划案、配置表、功能实现等），每个子需求明确负责人和预估工时。

✅ 当前已有实践：家族系统已使用三级嵌套结构（总单→策划总单→策划案/配置表），CBT5 将此模式推广到所有系统。

2 开发阶段 — 代码可追溯

TAPD + SVN 联动：开发在 TAPD 中将需求状态更新为"开发中"，SVN 提交消息强制包含 --story=ID 或 --bug=ID。通过 SVN pre-commit hook 自动校验格式，TAPD 端自动关联代码变更记录。

⚠️ 当前问题：仅 7% 提交有 TAPD 关联。需要 hook + 团队培训双管齐下。

3 测试阶段 — 缺陷闭环

TAPD操作：开发完成后需求状态 → "转测试"。测试在 TAPD 中提单关联对应需求。缺陷按 SLA 流转：致命 4h / 严重 24h / 一般迭代内。验证通过后缺陷 → "已关闭"、需求 → "产品体验" → "已关闭"。

4 发布阶段 — 质量门禁

TAPD看板：迭代结束前创建发布 Checklist 需求单，逐项确认：所有 P0 缺陷关闭、核心模块回归通过、IOS/Android 兼容性验证完成。版本发布后在 SVN 创建 release 分支打 tag。
