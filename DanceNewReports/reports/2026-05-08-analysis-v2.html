🏗️ 底层架构优化与重构

1. Player 类体系重构

旧的 PlayerManager 单体架构被迁移为多类型 Player 体系：

  • BasePlayer (abstract) — 统一基类，提供 ID、初始化、Update、LateUpdate、Dispose 生命周期
  • ScenePlayer — 普通场景玩家（原 NONE 类型替代）
  • RoomPlayer — 房间玩家（原 ROOM 类型，带 DJ 台）
  • BattlePlayer — 战斗玩家（原 BATTLE 类型，带箭头和计分）
  • TimelinePlayer — Timeline 演出玩家（原 WEDDING 类型，用于婚礼/剧情）
  • RTTPlayer — 实时传输玩家（旁观者/观战）
  • PhotoPlayer — 照相馆玩家（原 PHOTO 类型，带点击触发）

2. ScenePlayerManager -> Singleton 化

旧版 PlayerManager 被注释，新 ScenePlayerManager 采用 Singleton<T> 模式，通过 EventManager 驱动 MN_SCENE_UPDATE_PLAYER 事件来增删玩家。旧 PlayerManager 的代码以大段注释保留（约 200 行），供架构过渡参考。

3. 网络协议层扩展

  • S2CRoom.cs 扩展至 27454 行 — 房间协议（含照相馆子协议）

GuideManager.Init() 和 RedDotManager.Instance.Init() 自上一阶段起持续注释，这意味着新手引导系统和红点提示系统完全不可用。对于一款面向新玩家的社交游戏，缺少新手引导将严重影响用户留存。建议尽快修复。

🔧 技术深度解析

1. 舞蹈战斗的 P 点判定架构

BattleRenderManager 对外暴露三个事件驱动管道：

  • MN_BATTLE_HIT_EFFECT — 命中特效 (id, 结果, 连击数) → 触发粒子/颜色反馈
  • MN_BATTLE_HIT_ACTION — 命中动画 (id, 结果, 成功动画, 失败动画) → Perfect 触发 TAction 加速播放，其他结果触发 FAction
  • MN_BATTLE_FINISH_ACTION — 完成动画 (id, 模式, 阵营) → 按胜负播放 WinAction/LoseAction

这种事件驱动的设计将 判定逻辑 (BattleManager) 与 表现逻辑 (BattleRenderManager) 解耦，便于支持多种玩法模式（单人/双人/篮球场）。

2. Timeline 婚礼演出的"剧本"模式

婚礼系统采用服务端驱动的"剧本"模式：

  • 服务端通过 Timeline 协议下发剧情节点（EWeddingEvent）
  • 客户端 TimelineManager 监听事件，按需加载/播放不同 Timeline 资源
  • Timeline 资源内嵌自定义轨道（WeddingTrack）：EventClip（触发事件）、CreateClip（创建角色）、MoveClip（移动角色）、AnimClip（播放动画）
  • WeddingManager 管理和驱动的 TimelinePlayer 直接嵌入 Timeline 时间线

3. 照相馆的实时同步架构

PhotoStudioManager 实现了全双工实时同步：

  • 动作同步：C2SRoomPhotoPlayerModifyMontion → 服务端广播 → 全部客户端映射到 PhotoAnimaPair 定义
  • 位置同步：C2SRoomPhotoPlayerModifyTransition → 服务端广播 → 客户端 Lerp 插值
  • 相机同步：C2SPhotoRoomCamera → 服务端转发 → 非房主客户端接收 PhotoCameraTransitionEsp
  • 装备预览：C2SPhotoRoomChangeClothes → 服务端通知 → ReloadingEquipMaskObject 遮罩切换

4. HybridCLR 热更新架构

12 个热更新 DLL 与项目 csproj 对齐：Config (28.3KB) + GameCommon (125KB) + Proto (307KB) + Manager (147KB) + DataManager + Character + Battle + GamePlay + Net + Scene + UI + Timeline。所有新增的业务逻辑（WeddingManager、TimelineManager、BattleRenderManager）均位于 GamePlay.dll 热更新层，支持线上热修。
