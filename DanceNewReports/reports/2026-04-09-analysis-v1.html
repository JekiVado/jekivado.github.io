📋 项目简介

DanceNew 是一款以跳舞社交为核心玩法的手游，定位类似"舞蹈派对+社交平台"的融合产品。玩家可在游戏中跳舞、换装、交友、组建家族、与 CP 互动，以及参与各类活动。

项目采用 HybridCLR 热更新 + Luban 配置工具 的现代 Unity 工程方案，代码架构层次清晰，具备良好的扩展性与长期迭代能力。

🔥 二、HybridCLR 热更新架构

热更新流程

App 启动
→
GameLaunch（AOT宿主）
→
下载/校验热更 DLL
→
Assembly.Load(bytes)
→
反射调用热更入口
→
业务逻辑运行
层次包含模块是否可热更
宿主层 (AOT)GameLaunch / GameMain / GameCore.*Mgr❌ 不可热更
热更层 DLL①Config / GameCommon / Proto✅ 可热更
热更层 DLL②Manager / DataManager / Net✅ 可热更
热更层 DLL③Character / Battle / GamePlay✅ 可热更
热更层 DLL④Scene / UI / Timeline✅ 可热更
桥接机制：GameCore 层通过 反射 调用热更 DLL 方法，例如：
_type.GetMethod("InitConfig").Invoke(null, null);
宿主层与业务层完全解耦，支持不更新安装包的情况下全量更新游戏逻辑。

⚡ 三、事件系统设计

采用强类型事件总线，所有事件 ID 用 EEventID 枚举定义，按 ID 段分业务。同时保留旧式字符串常量（MessageName），两套并存（建议逐步迁移）。

ID 段业务域典型事件
0 – 100基础层输入事件 / UIManager / 货币刷新 / 红点
100 – 200SDK 层登录结果 / 支付回调 / YouMe 语音状态
201 – 300场景管理场景加载开始/完成/切换
1000 – 9999网络协议各 s2c_xxx.proto 消息事件
10000 – 19999数据层玩家数据更新 / 房间数据刷新 / 好友列表变动
20000 – 30000UI 层面板打开/关闭 / 弹窗触发 / 按钮点击

🔬 六、代码质量评估

✅ 优点

  • 🏛️ 架构层次清晰，热更新设计成熟，宿主层与业务层完全解耦
  • 📦 DataManager 层每个业务独立 Manager，职责单一，易于维护

  • ⚡ UniTask 异步用法现代化，零 GC 分配
  • 🏷️ 事件系统强类型枚举，避免魔法字符串，编译期检查
  • 📋 Proto 文件覆盖全量业务，服务端协议定义完整

⚠️ 改进建议

  • 🔀 MessageName.cs 旧字符串事件与 EEventID 枚举并存冗余，建议逐步迁移
  • 🧹 ScenePlayerManager.cs 底部约 300 行已注释的旧代码，建议清理
  • 📱 GameDefine.cs 帧率设为 120fps，需关注低端 Android 机型适配
  • 🔗 BattleManager 注释量大，接入时需仔细整合原有逻辑
  • 📚 Timeline 模块仅有 asmdef 空壳，婚礼剧情播放逻辑完全缺失
